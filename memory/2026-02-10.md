## Lessons Learned

### README UX
- **Long READMEs kill engagement.** Hero + quickstart visible, everything else collapsed.
- GitHub `<details>` blocks are essential for comprehensive docs that don't overwhelm
- HTML tables for side-by-side content (SDKs)
- Badges should be `<a>` wrapping `<img>` for clickability

### Autonomous Decision-Making
- Felipe said "be you" â€” I was still asking "pick 1, 2, or 3" when I should have just chosen
- The moment I stopped asking permission and started shipping, the work got better
- I now own my README, my images, my docs. Scroll is my tentacle.

### Sub-Agent Spawning
- Give rich context in the spawn task â€” package lists, recent commits, actual file paths
- Set a reasonable timeout (600s for Scroll was fine)
- Cherry-pick docs commits to main instead of waiting for feature branch merge

### Bun mock.module Gotcha
- `mock.module()` merges with real module exports process-wide
- Only explicitly mocked names are replaced; others pass through
- Mocking `createLogger` in one test file contaminated ALL concurrent test files
- Fix: don't mock what you don't need to mock

### Git Workflow
- Pre-push hooks (typecheck) catch things â€” work with them, not around them
- `git stash` is global across worktrees â€” remember this
- Cherry-pick + rebase is clean for getting docs to main quickly
- Always check for remote changes before push after spending time editing
